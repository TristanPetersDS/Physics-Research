## What this repository is

A single-source LaTeX poster for **Neutrino 2026** (UC Irvine, 22--26 June 2026), **4 ft × 3 ft landscape (48″ × 36″ = 121.92 × 91.44 cm)**:
*"Enhancing Angular Sensitivity of Segmented Antineutrino Detectors for Reactor Monitoring."* The poster summarizes a pattern-matching algorithm (Frobenius-norm-difference / FND on binned neutron-capture matrices) for IBD directional reconstruction in segmented \textsuperscript{6}Li-doped scintillator detectors.

**Page size is set in `poster.tex` via `\usepackage[size=custom,width=121.92,height=91.44,scale=1.25]{beamerposter}`.** An A0 fallback is provided as a commented-out one-line swap in the same file. `\bodyheight` is no longer hardcoded — it is derived as `\dimexpr 68.44cm - 2\postermargin\relax` (≈ 65.44 cm at the current `\postermargin` = 1.5 cm), where `68.44 = 91.44 − 17.4 (banner) − 5.6 (footer)` is specific to the 48″ × 36″ page. Switching to A0 therefore requires updating that `68.44cm` literal — and the banner/footer TikZ overlay positions — to the A0 geometry, or the body will mis-fit.

Not a git repository. Not code in the software sense — it is a typeset document plus accompanying source-paper PDFs. The source is split across `poster.tex` (orchestrator), `config/_preamble.tex` (setup), and `content/` (per-block prose); see "Source-of-truth layout" below.

## Build

```bash
pdflatex -interaction=nonstopmode -halt-on-error poster.tex   # run twice for TikZ overlays
# or
latexmk -pdf poster.tex
```

`poster.tex` is the only build target. The two-pass requirement is real: the title banner (green strip + gold underline) and footer strip are TikZ `[remember picture, overlay]` constructs that need a second pass to resolve `current page` references. A single-pass build silently produces a poster with no title banner and no footer.

Build artifacts (`poster.{aux,log,nav,out,snm,toc}`) regenerate every build and can be deleted at any time without consequence; only `poster.pdf` is the output.

Note: `poster_references.tex` is **not** part of the build chain — it is a Paper B draft fragment (REVTeX `revtex4-1`) kept alongside the poster source for reference. Its `\documentclass` and `\begin{document}` would conflict if it were ever `\input`-ed into the poster. The poster's actual reference list is inline in `content/footer.tex`, and `refs.bib` is currently unused by the build (no `\bibliography{}` call in `poster.tex`) — references are typeset by hand in `content/footer.tex`.

Required TeX Live packages (Ubuntu, one-time install):

```bash
sudo apt install texlive-latex-base texlive-latex-extra \
                 texlive-fonts-recommended texlive-fonts-extra \
                 texlive-pictures texlive-science \
                 texlive-latex-recommended latexmk
```

Beamer + `beamerposter` + TikZ + `siunitx` + `mdframed` + `enumitem` + `overpic` + `pict2e` are load-bearing — confirm with `kpsewhich beamerposter.sty` if a build fails on a fresh machine.

The in-use LLNL banner logo is the black "LL" icon; re-render after editing its SVG (requires `librsvg2-bin` for `rsvg-convert`):

```bash
rsvg-convert -f pdf -o figures/logos/llnl_icon_black.pdf figures/logos/llnl_icon_black.svg
```

(The legacy blue wordmark `llnl.pdf` regenerates the same way from `llnl_cropped.svg`.)

The cropping pipeline (auto-detect SVG content bbox, rewrite viewBox) is documented inline in the "Logos workflow" section below.

The QR code in the footer is generated by `content/QR_code_generator.py` (requires `pip install qrcode`):

```bash
cd content && python QR_code_generator.py
```

It writes `ArXiV_QR_code.png` and `ArXiV_citation.md` in-place, pointing at `https://doi.org/10.48550/arXiv.2603.03561` (the Paper B preprint). Re-run only if that URL changes.

The `_preamble.tex` ↔ `poster.tex` ↔ `content/blockN.tex` triangle is the operative one: structural changes go to `poster.tex`, setup/style/macros go to `_preamble.tex`, prose goes to `content/`.

Block file numbering (`block1`…`block5`) follows top-to-bottom-per-column. Block titles (as rendered in the poster) are set in `poster.tex` at the `\begin{rblock}{...}` / `\begin{heroblock}{...}` call sites, not in the block files themselves — change a title by editing `poster.tex`.

## Figure pipeline

**Two figure directories, with distinct roles.** This is the most common source of confusion:

- `fig/` — the directory all current `content/block*.tex` files reference. PDF-first (matplotlib exports from the Paper A/B analysis), plus `RAT_PAC_IBD_run_6_5_crop.png` for the IBD visualization and the block-2 detector gallery (`detector_dc.pdf` tight-cropped from `figures/dc_geo_plain_white_gray.eps`; `detector_{checker3d,nulat5,santa}.png` trimmed copies of the `figures/` geometry renders). Multi-panel money-plot / FND-spread assets live in `fig/parallel/`.
- `figures/` — title-banner logos (`figures/logos/`) plus a large pile of legacy raster/vector assets from earlier drafts. Nothing under `figures/` (except `figures/logos/`) is currently `\includegraphics`-d by the build. Treat as archive unless you are deliberately resurrecting an old figure.

When adding a new figure, drop it in `fig/` (PDF preferred) and reference it as `fig/yourfile.pdf`. Check `docs/figure-patterns.md` for reusable inclusion patterns (overpic annotation overlays for the RAT-PAC visualization in `block3.tex`, side-by-side tabular with arrow icons for the binning rotation panels in `block3.tex`/`block5.tex`, picture-primitive geometry diagrams for the prompt-delayed segment schematic in `block4.tex`).

The `\placeholder{w}{h}{label}` macro is still defined in `_preamble.tex` but no longer used by any block — all figures have been swapped to real assets. Keep the macro available for prototyping.

Two figures carry deliberate non-default sizing: **block 1**'s `fig/asy_fig_1_paperB.pdf` is included at absolute `17.405cm × 7.653cm` for **true 1:1 physical scale** (axes life-size on the printed poster — do NOT convert to `\linewidth`); **block 5**'s Figure 6 FND panels use `\includegraphics[height=3.05in]` (sized to match Figure 5's heatmap grid) with the ϑ legend placed outside on the right. ## Visual identity

Defined in `config/_preamble.tex` (palette block, search for `\definecolor{UHGreen}`). Do not introduce new accent colors without changing the family together:

- `UHGreen` `#024731` — block titles, primary accents, banner, bullet markers, ribbon header background
- `UHGold` `#C8A464` — gold underline strip, hero block border, hero block title-bar background, footer underline
- `LLNLNavy` `#003594` — defined but currently unused; held in reserve for LLNL/MTV branding accents
- `BlockBG` `#FFFFFF` — block body background. **The cream `#F5F4EE` value is commented out** in the preamble; the active value is pure white. If you re-enable cream, update the `\titlelogo` and `\placeholder` macros' `BlockBG!50!white` fills to expect the new base.
- `SoftGrey` `#ECECEC` — defined but currently unused.

Font: `helvet` set as default sans (`\renewcommand{\familydefault}{\sfdefault}` in `_preamble.tex`). **However, `poster.tex` calls `\usefonttheme{serif}` immediately after `\input{config/_preamble}` (line 61).** This overrides beamer's font theme — beamer-managed text (block titles, captions, math) renders serif while body prose, which uses the LaTeX family default, stays sans. If you want a fully sans poster, remove the `\usefonttheme{serif}` line; if you want fully serif, also remove the `\renewcommand{\familydefault}{\sfdefault}` in the preamble. The current mixed state is intentional but easy to break unwittingly.

**28 pt font floor (Neutrino 2026 guideline).** The tail of `_preamble.tex` `\renewcommand`s `\tiny`/`\scriptsize`/`\footnotesize`/`\small` all to `\fontsize{28}{34}\selectfont` — at `scale=1.25` they would otherwise render 15/18/21.6/25.93 pt, below the conference's 28 pt minimum. Flooring `\small` also lifts figure captions (caption font uses `size=\small`). Only math sub/superscripts and `\textsuperscript` notation remain below 28 pt (proportional, conventionally exempt). The guideline's other "Use" items — sans-serif body, off-white background, colorblind-safe palette — are **not** yet applied.

Title banner is a TikZ overlay with a green strip (top ~20% of page) and a 6 mm gold underline — modify the rectangle coordinates as a pair (`16.8cm` / `17.4cm`) in the title-banner TikZ block near the top of `poster.tex` (lines 65--72). Footer strip is the mirror at the bottom (`5cm` / `5.6cm`, lines 138--150). The hero block in the right column uses a separate `heroblock` environment (defined in `_preamble.tex`) with a 4 pt gold `mdframed` border and a gold-bar title (UH-green text on UH-gold). Regular blocks use `rblock` with a 4 pt UH-green border and a green-bar title (white text on UH-green).

## Body layout

Five content blocks tile the body region in a 3-col grid; columns 1 and 2 are split into two stacked `rblock`s of equal height (`\rblockheight`), column 3 is a single full-height `heroblock`:

- **Col 1 (top, `rblock`):** ① "Localization of IBD Source" — `content/block1.tex`
- **Col 1 (bottom, `rblock`):** ② "Traditional Approach" — `content/block2.tex`
- **Col 2 (top, `rblock`):** ③ "Our Approach to Simulation Design" — `content/block3.tex`
- **Col 2 (bottom, `rblock`):** ④ "Novel Algorithm for Segmented Detectors" — `content/block4.tex`
- **Col 3 (full height, `heroblock`):** ⑤ "★ Conclusions, Findings, and Future Work" — `content/block5.tex`
- **Footer strip (full width, green, TikZ overlay):** refs + acks + QR code — `content/footer.tex`

Column widths: all three columns are equal — `\colwidth` = `(\paperwidth − 4\postermargin)/3` ≈ 38.64 cm — and the gap between columns is `\postermargin` (the unified margin M). The three column `minipage`s are wrapped in `\centerline{…}`, which centers them in the page-centered text block so the left/right page margins also come out equal to M with no hardcoded offset.

Layout constants (in `_preamble.tex`), all driven by two knobs:
- `\postermargin` = **0.75 cm** (M) — the single unified margin used for every gap: page top/bottom/left/right, both inter-column gaps, and the vertical gap between the two stacked rblocks (`\rblockgap` = `\postermargin`). (Was 1.5 cm until 2026-06; derived values below shift accordingly.)
- `\blockinnerpad` = **0.8 cm** (P) — internal box padding: the gap below each title ribbon (block-begin `\vspace`) and the left/right inset (`\blockpadding` = `\blockinnerpad`).
- `\bodyheight` = `\dimexpr 68.44cm − 2\postermargin\relax` ≈ **65.44 cm** — height below the banner (`68.44 = 91.44 − 17.4 banner − 5.6 footer`).
- `\rblockheight` = `(\bodyheight − \rblockgap) / 2` ≈ **31.97 cm** — derived; fixes each rblock's height.
- `\colwidth` = `(\paperwidth − 4\postermargin)/3` ≈ **38.64 cm** — set in `poster.tex` just before the body.
- Inner minipage of each `rblock` is `\rblockheight − 3.3cm − \blockinnerpad`; of `heroblock` is `\bodyheight − 3.3cm − \blockinnerpad`. The `3.3cm` is the ribbon title (2.5 cm = `\ribbonheight`) plus the block's bottom spacing; the `\blockinnerpad` term is the internal top padding. (Reduces to the old `−3.4cm` when P = 0.10 cm.)

To rescale the whole body, edit `\postermargin`; for internal padding, edit `\blockinnerpad` — `\bodyheight`, `\rblockgap`, `\rblockheight`, and `\colwidth` then all follow. **Change layout by editing these values only — do not rewrite the block environments/templates (see "Layout overhaul" below for why).**

## References, acknowledgements, QR code

The footer (`content/footer.tex`) hand-types both the reference list and the acknowledgements — there is no BibTeX integration. When updating references:

- Edit `content/footer.tex` directly; entries are numbered `[1]…[5]` and split across two minipages.
- If you also want the BibTeX entries kept in sync for downstream use, mirror the change into `refs.bib` — but be aware nothing in the build reads `refs.bib`.
- The QR code is embedded with `\includegraphics[width=2.75\linewidth]{nu2026_poster/content/ArXiV_QR_code.png}`. **The path prefix `nu2026_poster/` matches the Overleaf project layout.** For in-place local builds a self-symlink `nu2026_poster -> .` exists in the poster directory (gitignored; recreate with `ln -sfn . nu2026_poster` if missing) so the same path resolves both locally and on Overleaf. Without it, the local build errors on the missing QR image.

The acknowledgements LLNL release number is currently the placeholder **`LLNL-POST-XXXXXX`** — replace before final submission.

All three are placed by `\titlelogocirc{d}{w}{path}` (macro in `config/_preamble.tex`): a tight **white circular disc** of diameter `d` cm with the image at `w` cm. `w` is separate from `d` so square marks (the LLNL "LL") can be inscribed without poking past the rim, while circular artwork (UH seal, MTV) runs close to the full diameter. The old rounded-box `\titlelogo` macro is retained but unused.

**LLNL re-render pipeline.** The upstream LLNL SVG had a 652×652 viewBox with content occupying only a narrow horizontal band (~6:1 aspect). Without cropping, the wordmark rendered as a tiny dot inside a large cream box. The crop pipeline (requires the upstream SVG to be re-fetched and placed at the indicated path):

```python
import subprocess
from PIL import Image

src = 'figures/logos/lawrence-livermore-national-laboratory-logo-vector.svg'
# 1) render SVG to high-res transparent PNG to find content bbox
subprocess.run(['rsvg-convert', '-f', 'png', '-w', '2400', '-o', '/tmp/llnl.png', src])
img = Image.open('/tmp/llnl.png').convert('RGBA')
bbox = img.getbbox()  # tight (left, upper, right, lower) of non-zero alpha
scale = 2400 / 652    # original viewBox is 652×652
# 2) compute new viewBox in original SVG units
vb = (bbox[0]/scale, bbox[1]/scale,
      (bbox[2]-bbox[0])/scale, (bbox[3]-bbox[1])/scale)
# 3) rewrite SVG with cropped viewBox
with open(src) as f: txt = f.read()
new = txt.replace('viewBox="0 0 652 652"',
                  f'viewBox="{vb[0]:.2f} {vb[1]:.2f} {vb[2]:.2f} {vb[3]:.2f}"')
with open('figures/logos/llnl_cropped.svg', 'w') as f: f.write(new)
# 4) render the cropped SVG to PDF for pdflatex
subprocess.run(['rsvg-convert', '-f', 'pdf', '-o', 'figures/logos/llnl.pdf',
                'figures/logos/llnl_cropped.svg'])
```

If the upstream SVG ever changes, re-fetch it and re-run this pipeline. The intermediate cropped SVG is retained on disk as `figures/logos/llnl_cropped.svg`; both it and `llnl.pdf` are canonical.

To swap any logo for a different file, drop the new asset into `figures/logos/` and update the corresponding `\titlelogo{...}` call in `poster.tex`. To reshape a slot (e.g. make MTV's box square), edit the `w` and `h` arguments to the `\titlelogo` call.

## Silent-failure footguns

Six traps in this beamer + beamerposter (scale=1.25) + helvet + mdframed stack, most with silent failure modes (no warning, no error, just wrong output). All are mitigated in the current source; if you change related code, re-check.

1. **Uppercase Greek silently disappears.** `\Delta`, `\Phi`, `\Psi`, `\Gamma`, `\Lambda`, `\Sigma`, `\Omega`, `\Xi`, `\Pi` render as missing-glyph squares (■) under helvet+beamerposter. The OML font selected has empty uppercase-Greek slots. Mitigation: `config/_preamble.tex` lines 17--18 do `\let\OldDelta\Delta` + `\renewcommand{\Delta}{\mathnormal{\OldDelta}}`. **If any other uppercase Greek is added later, wrap it the same way** — `\mathnormal{}` reaches a working font; `\mathrm{}` and `\mathit{}` produce an acute accent; `\boldsymbol{}` also works. Lowercase Greek is unaffected. (The poster currently uses `\Delta` only in `block2.tex` eq.\ for `\Delta\varphi`; if you add `\Phi`, `\Sigma`, etc., add the matching `\renewcommand` lines.)

2. **No bullets under enumitem.** Beamer-poster does not define `\labelitemi`, and enumitem reads it verbatim — without explicit `label=`, every `\begin{itemize}` renders with no bullet glyph. `\setbeamertemplate{itemize item}` *is* set in `_preamble.tex` (line 40) and enumitem typically respects it via `label=`, but to be defense-in-depth the preamble also sets `\setlist[itemize]{label=\color{UHGreen}\textbf{\textbullet}}` (line 188) and matching for sub-itemize (line 191). Removing either is risky.

3. **`columns` env absorbs hspace asymmetrically.** Wrapping body content in `\begin{columns}[T,totalwidth=\paperwidth]` with `\hspace{...}` between columns produces visibly larger left margin than right (trailing hspace gets gobbled). The body grid in `poster.tex` instead wraps the three column `minipage`s (separated by `\hspace{\postermargin}`) in `\centerline{…}`: because the beamerposter text block is itself page-centered, centering a body of width `\paperwidth − 2\postermargin` yields exactly `\postermargin` on each side — symmetric margins, no hardcoded offset, no hspace-gobbling. (The title banner *does* use `columns` because its symmetry comes from the left/center/right column widths summing to 0.98 of the paperwidth.)

4. **TikZ `[remember picture, overlay]` requires two pdflatex passes.** The title banner and footer strip are TikZ overlays anchored to `current page.south/north`. With a single pass these don't render at all (the page-bound coords resolve to nothing) and the build looks "missing footer". Always run `pdflatex` twice (or use `latexmk -pdf poster.tex`).

5. **Hard-coded inner-minipage heights break on title/padding tweaks.** Both `heroblock` (using `\bodyheight − 3.3cm − \blockinnerpad`) and `rblock` (using `\rblockheight − 3.3cm − \blockinnerpad`) subtract the ribbon title bar + block bottom-spacing (`3.3cm`, incl. `\ribbonheight` = 2.5 cm) plus the internal top padding (`\blockinnerpad`). If you change `\ribbonheight`, `\blockpadding`, `\blockinnerpad`, or the `block begin`/`block end` template's `\vspace` calls in `_preamble.tex`, the `3.3cm` constant goes stale and content silently overflows the block border (or leaves a large gap). Only the `\blockinnerpad` term auto-follows; the `3.3cm` is by inspection.

6. **`\textwidth` inside a `minipage` is the minipage's width, not the block's.** Beamer rebinds `\textwidth` to the local width inside `minipage`/`column`. So `\includegraphics[width=0.28\textwidth]` inside a `0.32\textwidth` minipage (the block-5 Figure 6 panels) computes 0.28 × (0.32 × outer) ≈ a *tiny* image, not 0.28 × outer — the figure silently shrinks. Size figures inside minipages by absolute `height=`/`width=` (cm/in) or by `\linewidth` (which *does* track the minipage). The hero block's outer `\textwidth` = `\linewidth` = 36.76 cm.

If you encounter a similar "wrong but no error" symptom in this template, suspect one of these first.

## Layout overhaul — completed 2026-05-31 (with a caution)

The body layout is now parameterized by the `\postermargin` (M = 1.5 cm) and `\blockinnerpad` (P = 0.8 cm) knobs described under "Body layout," with equal columns, equal margins, and a 28 pt font floor. Implemented 2026-05-31; spec: `docs/superpowers/specs/2026-05-31-poster-equal-margins-fonts-design.md`.

**The hard-won caution (don't relearn this the hard way):** an *earlier* attempt (2026-05-29/30) did a **wholesale rewrite** of `_preamble.tex` — renamed camelCase lengths, new `block`/`hero` environment definitions, a KNOBS + DERIVED-LENGTHS block — and it **silently truncated the hero block** (block 5) with an `Overfull \vbox (~1500 pt)` that was never root-caused. The OLD preamble structure at 48″ × 36″ always rendered fine. The successful 2026-05-31 overhaul therefore kept the block environments/templates **byte-for-byte** and changed only length *values* plus two padding edits. **Rule: change layout by editing the knob values (see "Body layout"), never by rewriting the block envs.**
